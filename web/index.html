How It Works

Four stages turn a raw PDF into a verifiable, abstention-capable Q&A system.

1

Typed Graph Extraction

The PDF is chunked and each chunk is processed by an LLM to extract typed nodes: party, obligation, right, numeric fact, condition, definition, blank, and reference. Co-occurring nodes within a chunk are connected by edges. This replaces untyped embedding search with a structured knowledge graph.

2

Multi-Variant Retrieval

At query time, three parallel LLM calls generate query variants (paraphrase, type-focused, clause-focused) and retrieve candidate clauses for each. A consensus filter keeps only clauses appearing in two or more variants, reducing noise from any single retrieval failure.

3

Answerability Gate

Two independent signals vote: (a) an LLM judges whether the retrieved evidence actually answers the question; (b) a topology scorer measures graph connectivity of retrieved nodes — high connectivity means the document genuinely covers the topic. Only when both signals pass does the system proceed to generate an answer.

✓ ✗

Grounded Answer or Principled Abstention

If the gate passes, an LLM generates an answer grounded strictly on the verified evidence set, citing the exact clause. If the gate fails, the response is "unanswerable" with a reason — no hallucination pathway exists in the design.

Technical Notes

Graph construction

Contracts are chunked with overlap, then each chunk is passed to an LLM that extracts typed nodes. Node types: definition, obligation, right, numeric, condition, blank, reference. Edges connect co-occurring nodes within the same chunk. The resulting graph is stored and reused across all queries on the same document.

Retrieval stability via query variants

A single query can be ambiguous or phrased in ways that miss relevant clauses. Three variants (paraphrase, type-focused, clause-focused) each run retrieval independently. Only clauses appearing in two or more results pass the consensus filter, reducing variance from individual retrieval failures.

Topology scoring

Beyond LLM judgment, a topology signal quantifies how well-connected the retrieved nodes are in the graph. High connectivity (short paths, dense neighborhoods near retrieved nodes) correlates with the document genuinely covering the question topic. The threshold is calibrated relative to document size.

Answerability decision logic

The final gate combines two boolean signals: (1) LLM-judged answerability given the retrieved evidence; (2) topology score above the calibrated threshold. Both must pass. This reduces false positives (answering when we shouldn't) more aggressively than either signal alone.

About

Deterministic RAG is a student research project building a legal AI system that is trustworthy by design: grounded retrieval, graph-verified evidence, and principled abstention when the document does not support an answer. It targets the hallucination problem that makes current legal AI unreliable in practice, where confident wrong answers carry real consequences.
