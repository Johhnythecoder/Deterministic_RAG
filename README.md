# Deterministic RAG: Answerability-Aware Contract QA

A system that determines whether a question is actually answerable from a document before responding.

Instead of returning text that only looks relevant, this system verifies whether the evidence truly supports an answer. If it does, it returns supporting text. If not, it abstains.

## Live Demo

👉 https://deterministic-rag-demo-p2jyg.ondigitalocean.app/

## Key Idea

Most RAG systems retrieve content that is topically similar, but not necessarily correct.

This system focuses on a different question:

Is the answer actually supported by the document?

It combines:
- LLM-based answerability judgment
- Graph-structured document evidence

These signals determine whether there is enough support to answer.

## Results

- ~87% answerable detection accuracy
- ~98% unanswerable detection accuracy
- Evaluated on 500+ questions across 12+ real contracts

## Example

Answerable  
Q: Who owns the Output generated by the Services?  
→ Returns answer + supporting text

Unanswerable  
Q: What is the monthly subscription price?  
→ Refuses to answer

## Why This Matters

In legal settings, a confident wrong answer creates real risk.

This system is designed to only answer when evidence is actually present, and refuse when it is not.

## How It Works (High Level)

1. Extracts structured nodes (obligations, definitions, etc.) from the document
2. Retrieves evidence using multiple query variants
3. Evaluates whether the evidence is sufficient to answer
4. Either returns a grounded answer or abstains

## Notes

- Demo-focused system, not production optimized
- Single-document Q&A, with multi-document expansion as future work
- Processing can take time because the full pipeline runs
